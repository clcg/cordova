get_db_version_num()

get_db_version_num(boolean $use_point) : int

Get DB Version Num

Returns the version number for the latest database if $config['vd_version'] is set to 0. Otherwise the specified version will be used.

For point releases, the name of a table will be like 'dvd_2_1', not 'dvd_2.1'. Therefore, this function returns the underscored format (e.g. 'dvd_2_1') by default. To return the point format (e.g. 'dvd_2.1'), specify the first parameter to be TRUE.

Parameters

boolean $use_point

Return a point (.) format instead of a underscore (_) format

Returns

int —

Correct version of the variation database

get_new_version_name()

get_new_version_name(string $cur_name, int $latest_version) : string

Get New Version Name

Returns the name of the next version of a table based on the current name.

For example: if current name = 'dvd_1' then next name = 'dvd_2'

The latest version number is computed automatically. However, you can specify your own latest version using the second parameter.

Parameters

string $cur_name

Current name of table
int $latest_version

Latest version number

Returns

string —

Next version of a table name

get_ids_within_range()

get_ids_within_range(string $table, int $start_pos, int $limit) : array

Get IDs Within Range

Returns an array of IDs within a range of rows in the database table. This is good for pagination.

For example, if you specify your range as get_ids_within_range($this->tables['vd_queue'], 0, 10) You will get the variant IDs for the first 10 rows of the queue table (rows 0-9).

Parameters

string $table

Database table name
int $start_pos

Starting row
int $limit

Number of rows total (from/including the starting row)

Returns

array —

Unique IDs
